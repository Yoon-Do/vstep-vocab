## 💡 Key Insights Summary

1. **Writing is Special**: Writing tasks (writing-task1, writing-task2) are treated differently - forced list-only view, example boxes always shown, gradient styling

2. **Smart Detection**: Single function `isListOnlyTheme()` at app.js:182 determines if a theme uses list-only mode

3. **Data Structures**: All 70+ writing templates follow same format: `{ en, vi, type: "study", example }`

4. **Display Pipeline**: Theme selection → getVisibleWords() → renderList() → switchView() → DOM update

5. **4-Column Layout**: Grid defined in CSS at styles.css:410-411 creates: EN+theme | VI+example | metadata | actions

6. **No Routing**: Theme selection (not URL) drives everything - state.activeTheme is the key

7. **Persistence**: localStorage keeps user progress (mastered, favorites, settings)

8. **Responsive**: Adapts from desktop sidebar to mobile drawer to mobile bottom nav
